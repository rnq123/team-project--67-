# Архитектура игры "Угадай число"

## Обзор архитектуры

Игра использует модульную архитектуру с разделением ответственности между классами и функциями.

## Компоненты системы

### 1. Основной модуль (game.py)

**Класс GuessNumberGame**
- Центральный класс игры
- Управляет игровым процессом
- Координирует взаимодействие между компонентами

**Основные методы:**
- `__init__()`: Инициализация игры
- `show_menu()`: Отображение главного меню
- `play_round()`: Один игровой раунд
- `run()`: Основной игровой цикл

### 2. Модуль рекордов (records.py)

**Класс RecordsManager**
- Управление таблицей рекордов
- Работа с файловой системой для сохранения результатов

**Основные методы:**
- `load_records()`: Загрузка рекордов из файла
- `save_records()`: Сохранение рекордов в файл
- `add_record()`: Добавление нового рекорда
- `display_records()`: Отображение таблицы рекордов

### 3. Конфигурационный модуль (config.py)

**Константы и настройки:**
- `DIFFICULTY_LEVELS`: Параметры уровней сложности
- `RECORDS_FILE`: Имя файла для сохранения рекордов

## Поток данных

1. Пользователь запускает игру
2. Загружаются рекорды из файла
3. Отображается главное меню
4. При начале игры:
   - Запрашивается имя игрока
   - Выбирается уровень сложности
   - Генерируется случайное число
   - Начинается игровой раунд
5. При завершении игры:
   - Если игрок выиграл, результат сохраняется
   - Обновляется таблица рекордов
   - Результаты сохраняются в файл
